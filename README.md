## Инструкция по развертыванию локальной языковой модели:
1) Перейдите на официальный сайт ollama.com/download и загрузите дистрибутив Ollama, соответствующий вашей операционной системе;
2) Откройте интерфейс командной строки (терминал, PowerShell или CMD) и выполните следующую команду: ```ollama pull qwen2.5:7b```
3) Если вы используете операционную систему на базе Linux, после установки может потребоваться ручной запуск сервиса. Для этого выполните в терминале команду: ```ollama serve```

## Инструкция по запуску приложения на локальном сервере:
**1.** Запуск фронтенда:  
**1.1** Перейдите на официальный сайт https://nodejs.org/en/download и загрузите дистрибутив Node.js, соответствующий вашей операционной системе;  
**1.2** Откройте папку, в которой находится vue.config.js и скопируйте путь к файлу [1]  
**1.3** Откройте интерфейс командной строки (терминал, PowerShell или CMD) и выполните следующие команды:  

```bash
# Замените путь на свой и выполните команды
PROJECT_PATH="ваш_путь_из_пункта_1"
cd "$PROJECT_PATH" && npm install && npm run serve
```
**2.** Запуск ml-сервиса:  
**2.1** Перейдите на сайт https://www.python.org/downloads/release/python-31314/ и загрузите дистрибутив Python 3.13.14, соответствующий вашей операционной системе;  
**2.2** Откройте папку, в которой находится app.py и скопируйте путь к файлу [2]  
**2.3** В папке [2] создайте файл config.py и укажите в нем переменные окружения:  
- SERPER_API_KEY — ключ Serper.dev для поиска.
- LOCAL_LLM_URL — URL локальной LLM, по умолчанию http://localhost:11434/v1/chat/completions.
- LOCAL_MODEL — имя модели для локальной LLM, по умолчанию qwen2.5:7b.
- FLASK_HOST — host Flask-приложения.
- FLASK_PORT — port Flask-приложения.
- FLASK_DEBUG — включает debug-режим при значении true.
2.4) Откройте интерфейс командной строки (терминал, PowerShell или CMD) и выполните следующие команды:

```bash
# Замените путь на свой и выполните команды
PROJECT_PATH="ваш_путь_из_пункта_2"
cd "$PROJECT_PATH" && pip install -r requirements.txt && python app.py

```
3) Веб-сервер запущен и доступен локально по адресу `http://<IP>:<PORT>`, указанному в консоли

## Описание
Веб-приложение для автоматического создания краткого аналитического обзора по выбранной или введенной пользователем теме с функциями локальной фильтрации и модерации контента.

*Функционал:*

- Выбор темы:  поддержка ручного ввода текста или выбор готового варианта из списка;
- Анализ контента: генерация краткого аналитического отчета по запросу;
- Обработка исключений: вывод предупреждения при блокировке небезопасного запроса;
- Адаптивный дизайн: встроенная поддержка светлого и темного режимов отображения интерфейса;


## Пример пользовательского сценария

**1.** На главном экране находится поле ввода и список тем, пользователь может:
- ввести интересующую тему
- выбрать тему из списка

**2.** Пользователь нажимает кнопку "Начать анализ"  

**3.1.** Через некоторое время страница обновляется, на ней будет отображаться краткий анализ по указанной теме  
  
**3.2.** Запрос может быть отклонён из-за содержания в теме недопустимого контента, в таком случае на экране появится окно с текстом "Запрос отклонён", после чего пользователь будет перенаправлен на главную страницу
